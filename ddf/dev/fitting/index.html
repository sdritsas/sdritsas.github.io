Fitting

The objective of this section is to cover geometric constructions using points and vectors. In particular, this section focuses on fitting shapes through points.

Circle to 3 Points

Provided with three points a, b, and c compute the circle that passes through the points. Assume that the points are not colinear.

""" Inputs
"""
a = Point3d( ax, ay, az )
b = Point3d( bx, by, bz )
c = Point3d( cx, cy, cz )
Solution

A circle from three three points is also known as the circumscribed circle of the associated triangle. Geometrically it can be constructed as the intersection of two lines perpendicular to the sides of the triangle at their mid-points.

''' Constructive Geometry
'''
def Circumscribe( a, b, c ):
    ''' Plane Basis at a
    '''
    u = b - a
    v = c - b
    n = Vector3d.CrossProduct( u, v )

    ''' Orthogonal Directions
    '''
    M = Vector3d.CrossProduct( n, u )
    W = Vector3d.CrossProduct( n, v )
    N = Vector3d.CrossProduct( M, W )

    ''' Line Mid Points
    '''
    m = 0.5 * ( a + b )
    w = 0.5 * ( b + c )

    ''' Intersection
    '''
    t = Vector3d.CrossProduct( w - m , M ) * N / ( N * N )

    o = w + W * t
    r = o.DistanceTo( a )

    return Circle( Plane( o, n ), r )

If the points are already expressed in relationship to a plane, or in other words they are 2D, then the solution can be computed by solving a system of equations. Note that the determinant will be zero if all three points are colinear.

''' Circle from 2D Points
'''
def Circumscribe( a, b, c ):
    ''' Determinants
    '''
    D = ( a.X * ( b.Y - c.Y ) +
          b.X * ( c.Y - a.Y ) +
          c.X * ( a.Y - b.Y ) ) * 2.0

    aa = a.X * a.X + a.Y * a.Y
    bb = b.X * b.X + b.Y * b.Y
    cc = c.X * c.X + c.Y * c.Y

    ''' Center and Radius
    '''
    x = ( aa * ( b.Y - c.Y ) +
          bb * ( c.Y - a.Y ) +
          cc * ( a.Y - b.Y ) ) / D

    y = ( aa * ( c.X - b.X ) +
          bb * ( a.X - c.X ) +
          cc * ( b.X - a.X ) ) / D

    o = Point3d( x, y, 0 )
    r = o.DistanceTo( a )
    return Circle( o, r )

If the points are in space, then it can be computed using vector products. Note that if all points are colinear then the normal vector of the basis plane will become the zero vector which will lead to divisions by zero.

''' Circle from 3D Points
'''
def Circumscribe( a, b, c ):
    u, v, w = a - b, b - c, c - a
    n = Vector3d.CrossProduct( u, v )

    uu = u * u
    vv = v * v
    ww = w * w
    nn = n * n

    A = -0.5 * vv * u * w / nn
    B = -0.5 * ww * v * u / nn
    C = -0.5 * uu * w * v / nn

    o = a * A + b * B + c * C
    r = 0.5 * ( uu * vv * ww / nn ) ** 0.5
    return Circle( Plane( o, n ), r )

Line to Points 2D

Provided with a list of 2D points [p0, p1, ..., pn] compute a line ( o, u ), where o is its origin and u is its direction vector, that best fits the points.

""" Inputs
"""
points = [
    Point3d( 0.0, 0.0, 0.0 )
    Point3d( 1.0, 0.0, 0.0 ),
    Point3d( 0.0, 1.0, 0.0 ),
    # ...
    Point3d( 1.0, 1.0, 0.0 )]
Solution

There are several statistical methods for fitting lines to sets of 2D points, with two most common being simple linear regression and orthogonal regressions. The difference between them is in the loss or error function being minimized, namely the vertical or orthogonal distances from the fitted line to the points. Both methods use the same preliminary steps presented below. The origin of the fitted line is computed via the mean of the points' coordinates.

''' Point Average
'''
def Average( points ):
    mean = Point3d.Origin
    for point in points:
        mean.X += point.X
        mean.Y += point.Y
    return mean * ( 1.0 / len( points ) )

For computing the direction of the line, the covariance matrix is constructed first. The covariance matrix captures the dependencies among coordinates, in the all combinations sense. It is computed by first subtracting the mean from each points and then summing their coordinate products. Finally, the sums of products are normalized.

''' Covariance Matrix
'''
def Covariance( points, mean ):
    xx, xy, yy = 0.0, 0.0, 0.0
    for point in points:
        x = point.X - mean.X
        y = point.Y - mean.Y

        xx += x * x
        xy += x * y
        yy += y * y
    n = 1.0 / len( points )
    return xx * n, xy * n, yy * n

Simple Linear Regression

Simple linear regression is, as the name states, the easiest method to fit a line through a set of points. The direction of the line is computed from the components of the covariance matrix. Note that the term yy is not used and that normalizing the covariance matrix elements is not strictly required since the direction is eventually normalized.

''' Simple Linear Regression
'''
def SimpleLinearRegression( points ):
    o = Average( points )
    xx, xy, yy = Covariance( points, o )
    u = Vector3d( xx, xy, 0 ); u.Unitize( )
    return o, u

Orthogonal Regression

Orthogonal regression computes the line which minimizes the sum of the projected distances from the points to the line. It is associated with the Principal Components Analysis which generalizes this concept in higher dimensions. The general idea is to extract an orthogonal basis from the covariance matrix, such that the basis vectors align with directions of variance in the data points. This leads to an eigenvalue problem, which for the case of line fitting it is easy to compute. Since the covariance matrix is 2x2, its eigenvalues can be computed by solving the characteristic polynomial which is quadratic.

''' EigenValues
'''
def EigenValues( xx, xy, yy ):
    """ Characteristic Polynomial
        λ * λ − ( xx + yy ) * λ + ( xx * yy - xy * xy ) = 0
    """
    t = xx + yy             #-- Trace
    d = xx * yy - xy * xy   #-- Determinant

    a = 1                   #-- Quadratic coefficients
    b =-t
    c = d
    d = b * b - 4 * a * c   #-- Discriminant

    """ EigenValues
    """
    o = ( -b - math.sqrt( d ) ) / ( 2 * a )
    i = ( -b + math.sqrt( d ) ) / ( 2 * a )
    return o, i

The origin of the regression line is derived in the same fashion as with simple linear regression. The direction is computed for the eigenvector associated with the largest eigenvalue. Note that depending of the order of points in the list the eigenvector may flip in the ±u sense.

''' Orthogonal Regression
'''
def OrthogonalRegression( points ):
    o = Average( points )
    xx, xy, yy = Covariance( points, o )
    e = max( EigenValues( xx, xy, yy ) )
    u = Vector3d( xy, e - xx, 0 ); u.Unitize( )
    return o, u

Rhino Line Fitting

Line to Points 3D

Provided with a list of 3D points [p0, p1, ..., pn] compute a line ( o, u ), where o is its origin and u is its direction vector, that best fits the points.

""" Inputs
"""
points = [
    Point3d( 0.0, 0.0,-1.0 )
    Point3d( 1.0, 0.0, 0.0 ),
    Point3d( 0.0, 1.0, 1.0 ),
    # ...
    Point3d( 1.0, 1.0, 2.0 )]
Solution

Fitting a line through 3D points is based on the PCA approach presented earlier. This is because there is no standard way to measure the loss other than in the orthogonal distance sense. The steps followed are pretty much the same: (1) compute the mean point, (2) construct the covariance matrix, (3) extract the eigenvector associated with the largest eigenvalue.

''' Point Average 3D
'''
def Average( points ):
    mean = Point3d( 0.0, 0.0, 0.0 )
    for point in points:
        mean += point
    return mean * ( 1.0 / len( points ) )

Note that the covariance matrix is now a 3x3 matrix. It is constructed exactly in the same way, that is by summing the coordinate products. The covariance matrix is symmetric. Therefore, only the upper or lower triangle, including the diagonal elements are needed.

''' Covariance Matrix 3x3
'''
def Covariance( points, mean ):
    xx         = 0.0
    xy, yy     = 0.0, 0.0
    xz, yz, zz = 0.0, 0.0, 0.0

    for point in points:
        d = point - mean
        xx += d.X * d.X
        xy += d.X * d.Y; yy += d.Y * d.Y
        xz += d.X * d.Z; yz += d.Y * d.Z; zz += d.Z * d.Z

    n = 1.0 / len( points )
    return ( xx * n,
             xy * n, yy * n,
             xz * n, yz * n, zz * n )

While in theory it is possible to solve the cubic characteristic polynomial of the covariance matrix for its eigenvalues, it is not too pleasant or numerically sound. Instead, since only the eigenvector with the largest eigenvalue is needed, it can be computed iteratively using the power iteration method. An initially arbitrary vector is repeatedly multiplied with the covariance matrix and then normalized. It takes only a few iterations before the vector stops changing.

''' Power Iteration Method
'''
def PowerIteration( covariance, count = 32 ):
    xx, xy, yy, xz, yz, zz = covariance
    u = Vector3d( 1, 1, 1 )
    for index in range( count ):
        v = Vector3d( u.X * xx + u.Y * xy + u.Z * xz,
                      u.X * xy + u.Y * yy + u.Z * yz,
                      u.X * xz + u.Y * yz + u.Z * zz )
        v.Unitize( )
        w = u - v #-- Change?
        if( w * w <= 1e-15 ): break
        u = v     #-- Continue
    return u

The line of regression is thus obtained using the three steps previously developed. Note that the code below omits accounting for the trivial cases, such as the point list containing fewer than two points and/or all points being identical etc.

''' PCA Regression Line
'''
def PcaLineRegression( points ):
    o = Average( points )
    c = Covariance( points, o )
    u = PowerIteration( c )
    return o, u

The TryFitLineToPoints( ) method in Rhino computes exactly the same orthogonal regression line as seen in the line through 2D points scenario.

'''

o = ou.From
u = ou.Direction; u.Unitize( )

Plane to Points

Provided with a list of 3D points [p0, p1, ..., pn] compute a plane ( o, n ), where o is its origin and n is its normal vector, that best fits the points.

""" Inputs
"""
points = [
    Point3d( 0.0, 0.0,-1.0 )
    Point3d( 1.0, 0.0, 0.0 ),
    Point3d( 0.0, 1.0, 1.0 ),
    # ...
    Point3d( 1.0, 1.0, 2.0 )]
Solution

There are at least two common ways to construct a fitted plane through points depending on how the 'as close as possible' concept is defined: (1) The least-squares method minimizes the sum of distances along the Z-axis to the plane and (2) the principal components analysis method minimizes the projected distances onto the plane.

Linear Least Squares

The Linear Least Squares method uses the explicit equation of the plane ax + bx + cz + d = 0. By rearranging the terms this becomes xa/c + yb/c + d/c = z. This can be interpreted as a system of equations Ax = b, where the rows of the A matrix are the point coordinates [p.X, p.Y, 1.0], the rows of b are [p.Z] and x are the plane's coefficients a/c, b/c and d/c. Since there are typically more rows than the minimum required, that is three, the system is solved using the pseudo-inverse x = (ATA)-1ATb. The lstsq( ) function of numpy can be used for computing the best fit plane.

''' Least Squares in NumPy
'''
def LeastSquaresPlane( points ):
    import numpy as np
    A = np.array( points, dtype=float )
    b = np.copy( A.T[2] ); A.T[2] = 1.0

    x = np.linalg.lstsq( A, b )[0]
    return x.tolist( )

    # ata = np.linalg.inv( np.dot( A.T, A ) )
    # x = np.dot( np.dot( ata, A.T ), b )
    # return x.tolist( )

''' Points to NumPy
'''
def ToNumPy( points ):
    return [[p.X, p.Y, p.Z]
        for p in points]

''' Plane from Coefficients
'''
def ToRhino( result ):
    from Rhino.Geometry import Plane
    a, b, d = result
    plane = Plane( a, b, -1, d )

Principal Components Analysis

The Principal Components Analysis computes the plane that best fits the points in the orthogonal sense. The plane's origin is the mean point and its basis vectors are the two largest eigenvectors. With numpy the eigenvectors and eigenvalues are produced by the eig( ) function. The results are not sorted so it requires a bit more work to do so.

''' PCA Plane in NumPy
'''
def PcaPlane( points ):
    import numpy as np
    A = np.array( points, dtype=float )
    o = np.mean( A, axis = 0 )
    v, U = np.linalg.eig( np.cov( ( A - o ).T ) )

    #-- Sort the eigenvectors by the eigenvalues
    #--
    UV = [( u, v ) for u, v in zip( U.T.tolist( ), v.tolist( ) )]
    UV.sort( key = lambda uv: uv[1], reverse=True )
    uvx, uvy, uvz = UV
    U = ( uvx[0], uvy[0], uvz[0] )
    v = ( uvx[1], uvy[1], uvz[1] )
    return o.tolist( ), U, v

''' Points to NumPy
'''
def ToNumPy( points ):
    return [[p.X, p.Y, p.Z]
        for p in points]

''' Construct the Best Fit Plane
'''
def ToRhino( result ):
    from Rhino.Geometry import Point3d, Vector3d, Plane
    o, ( x, y, z ), v = result

    o =  Point3d( o[0], o[1], o[2] )
    x = Vector3d( x[0], x[1], x[2] )
    y = Vector3d( y[0], y[1], y[2] )

    plane = Plane( o, x, y )

In case numpy is not available or preferred, it is possible to compute the best fit plane using the power iteration method presented in the 3D line fitting section. The normal of the best fit plane is the eigenvector associated with the smallest eigenvalue of the covariance matrix. By inverting the covariance matrix, the eigenvalues are also inverted, therefore the power iteration method can be used to extract the normal direction.

''' Invert Covariance Matrix
'''
def Invert( S ):
    xx, xy, yy, xz, yz, zz = S

    d = ( xx * ( yy * zz - yz * yz ) -
          xy * ( xy * zz - yz * xz ) +
          xz * ( xy * yz - yy * xz ) )

    mxx = ( yy * zz - yz * yz ) / d
    mxy = ( yz * xz - xy * zz ) / d
    mxz = ( xy * yz - yy * xz ) / d

    myy = ( xx * zz - xz * xz ) / d
    myz = ( xy * xz - xx * yz ) / d
    mzz = ( xx * yy - xy * xy ) / d

    return ( mxx,
             mxy, myy,
             mxz, myz, mzz )

''' PCA Plane
'''
def PcaPlane( points ):
    o = Average( points )
    C = Covariance( points, o )
    K = Invert( C )
    u = PowerIteration( K )
    return Plane( o, u )

Singular Value Decomposition

The Singular Value Decomposition is a factorization method that expresses a matrix as product of an orthogonal square matrix, a diagonal matrix of singular values and a rectangular matrix. It is closely related with the principal components analysis method but does not require to compute the covariance matrix. Additionally, the svd( ) method of numpy returns the vectors and singular values sorted.

''' SVD Plane in NumPy
'''
def SvdPlane( points ):
    import numpy as np
    A = np.array( points, dtype=float )
    o = np.mean( A, axis = 0 )
    U, s, V = np.linalg.svd( A - o )
    return o.tolist( ), V.tolist( ), s.tolist( )

''' Points to NumPy
'''
def ToNumPy( points ):
    return [[p.X, p.Y, p.Z]
        for p in points]

''' Construct the Best Fit Plane
'''
def ToRhino( result ):
    from Rhino.Geometry import Point3d, Vector3d, Plane
    o, ( x, y, z ), s = result

    o =  Point3d( o[0], o[1], o[2] )
    x = Vector3d( x[0], x[1], x[2] )
    y = Vector3d( y[0], y[1], y[2] )

    plane = Plane( o, x, y )

Circle to Points

Provided with a list of 3D points [p0, p1, ..., pn] compute a circle ( o, n, r ), where o is its origin, n is its normal vector and r is its radius, that best fits the points.

""" Inputs
"""
points = [
    Point3d( 0.0, 0.0,-1.0 )
    Point3d( 1.0, 0.0, 0.0 ),
    Point3d( 0.0, 1.0, 1.0 ),
    # ...
    Point3d( 1.0, 1.0, 2.0 )]
Solutions

There are several approaches for computing the best-fit circles from a set of points. The method implemented below are based on the work of (Chernov and Lesort, 2008) and (Al-Sharadqah and Chernov, 2009). The most basic method is the pure algebraic fit, where the sum of the distances squared from the points to the circle is minimized. This is often expressed as sum( ( x - xo ) ** 2 + ( y - yo ) ** 2 - r ** 2 ) ** 2, where x and y are the points' coordinates and xo, yo and r are the unknown circle's center and radius. Its partial derivatives with respect to xo, yo and r leads to a system of three linear equations, which can be solved easily.

In pure python, first the mean point is computed and the moments matrix which contains the coefficients of the linear system. Note that subtracting out the mean point aims at preventing the sum products from numerically exploding. Nevertheless, if there are too many points the coefficients will inevitably become huge.

''' Mean Point
'''
def Average( points ):
    mean = Point3d( 0.0, 0.0, 0.0 )
    for point in points:
        mean += point
    return mean * ( 1.0 / len( points ) )

''' Moments Matrix
'''
def Moments( points, mean ):
    sx, sy, sz = 0.0, 0.0, 0.0
    xx, xy, yy = 0.0, 0.0, 0.0
    xz, yz, zz = 0.0, 0.0, 0.0

    for point in points:
        x = point.X - mean.X
        y = point.Y - mean.Y
        z = x * x + y * y

        sx += x; xx += x * x
        sy += y; xy += x * y
        sz += z; yy += y * y

        xz += x * z
        yz += y * z

    return ( sx, sy, sz,
             xx, xy, yy,
             xz, yz )

The pure algebraic circle fitting process involves: (1) computing the mean point, (2) factoring the coefficients of the moments matrix, (3) solving the 3x3 system of equations, (4) deriving the circle's center and radius from the solution's coefficients, and (5) translating the circle back to the mean.

''' Determinant of 3x3 Matrix
'''
def Det3x3( xx, yx, zx,
            xy, yy, zy,
            xz, yz, zz ):
    return ( xx * ( yy * zz - zy * yz ) -
             yx * ( xy * zz - zy * xz ) +
             zx * ( xy * yz - yy * xz ) )

''' Solve Ax = b using Determinants
'''
def Solve3x3( xx, yx, zx, bx,
              xy, yy, zy, by,
              xz, yz, zz, bz ):
    d = Det3x3( xx, yx, zx,
                xy, yy, zy,
                xz, yz, zz )
    #-- Check for Zero Determinant!
    x = Det3x3( bx, yx, zx,
                by, yy, zy,
                bz, yz, zz )

    y = Det3x3( xx, bx, zx,
                xy, by, zy,
                xz, bz, zz )

    z = Det3x3( xx, yx, bx,
                xy, yy, by,
                xz, yz, bz )

    return x / d, y / d, z / d

''' Pure Algebraic Circle Fit
'''
def PureAlgebraicCircleFit( points ):
    o = Average( points )
    x, y, z, xx, xy, yy, xz, yz = Moments( points, o )

    n = len( points )
    B, C, D = Solve3x3( xx, xy, x, -xz,
                        xy, yy, y, -yz,
                        x,  y,  n, -z )

    xo, yo = -B / 2, -C / 2
    r = ( xo * xo + yo * yo - D ) ** 0.5
    return Circle( Point3d( o.X + xo, o.Y + yo, 0 ), r )

What motivates alternative methods is the problem of inaccuracy encountered using the pure algebraic method to fit small arcs. The hyperarrurate circle fitting method by (Chernov and Lesort, 2008) provides an excellent alternative. The process involves solving a generalized eigenvalue problem for which the article by (Ghojgh, Karray and Crowley, 2023) provides insights to help decipher.

''' Hyperaccurate Circle Fitting
'''
def NumPyCode( points ):
    import numpy as np
    P = np.array( points, dtype=float )
    o = np.mean( P, axis = 0 )

    ''' Moments Matrix Factorization
    '''
    x = P.T[0] - o[0]
    y = P.T[1] - o[1]
    z = x * x + y * y
    w = np.ones( len( P ) )
    A = np.array( [z, x, y, w] )
    U, S, V = np.linalg.svd( A )
    #-- Check for singularity!

    ''' Constraint Matrix
    '''
    z, x, y, w = np.mean( A, axis = 1 )
    N = np.array( [[8 * z, 4 * x, 4 * y,  2],
                   [4 * x, 1,     0,      0],
                   [4 * y, 0,     1,      0],
                   [2,     0,     0,      0]] )

    ''' Solve Eigenvalue Problem
    '''
    W = np.dot( np.dot( U, np.diag( S ) ), U.T )
    U, S, V = np.linalg.svd( np.dot( np.dot(
        W, np.linalg.inv( N ) ), W ) )
    z, x, y, w = np.linalg.solve( W, U.T[-1] )

    ''' Extract Circle's Parameters
    '''
    r = 0.5 * ( x * x + y * y - 4 * z * w ) ** 0.5 / abs( z )
    x = o[0] - 0.5 * x / z
    y = o[1] - 0.5 * y / z
    return x, y, r

def ToNumPy( points ):
    return [[p.X, p.Y, 0.0]
        for p in points]

def ToRhino( result ):
    from Rhino.Geometry import Point3d, Circle
    x, y, r = result
    return Circle( Point3d( x, y, 0 ), r )
